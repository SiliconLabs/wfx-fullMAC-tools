1 Common API {#wf200_common_api}
2 ============
3 
4 To interact with WF200, the host has access to a set of API which are exposed both in the split MAC case and in the full MAC case. To find the related driver functions, you can refer to the table below or to the \ref COMMON_API group.
5 
6 ## Common API commands
7 
8 | Commands | Value | Associated driver function | Description |
9 |------------------|-------|----------------------------|---------------------------------------------------------|
10 | HI_CONFIGURATION | 0x09 | ::wf200_send_configuration | Configure the device through the \ref wf200_pds |
11 | HI_SHUT_DOWN | 0x32 | ::wf200_shutdown | Send a request to shut down the internal power supplies |
12 
13 For each available command, there is an associated confirmation with the same ID value.
14 
15 ## Common API indications
16 
17 | Indications | Value | Description |
18 |------------------|-------|--------------------|
19 | HI_EXCEPTION_IND | 0xe0 | Exception |
20 | HI_ERROR_IND | 0xe4 | Error |
21 | HI_STARTUP_IND | 0xe1 | Startup indication |
22 | HI_GENERIC_IND | 0xe3 | Generic status |
23 
24 This indications have to be managed by the host depending on the application and use case.
